### 🔬 01 · Deep Packet Inspection Engine

</div>

<div align="center">

![Java](https://img.shields.io/badge/Java-0d0d14?style=for-the-badge&logo=openjdk&logoColor=ff6b6b)
![Multithreading](https://img.shields.io/badge/Multithreading-0d0d14?style=for-the-badge&logoColor=4cffee)
![Networking](https://img.shields.io/badge/Networking-0d0d14?style=for-the-badge&logoColor=4cffee)
![TLS/SNI](https://img.shields.io/badge/TLS%2FSNI-0d0d14?style=for-the-badge&logoColor=ff88ff)
![PCAP](https://img.shields.io/badge/PCAP-0d0d14?style=for-the-badge&logoColor=ff88ff)

</div>

<br/>

A **multithreaded network analysis engine** written in Java that operates directly on real network packet captures. This goes far beyond surface-level networking — it handles stateful flow tracking, encrypted traffic classification, and real-time analytics.

| Component | What It Does |
|:----------|:------------|
| 📦 **PCAP Parser** | Reads and decodes real network traffic capture files at the binary level |
| 🔁 **Flow Tracker** | Uses Five-Tuple hashing (src IP, dst IP, src port, dst port, protocol) to track every unique connection statefully |
| 🔐 **TLS SNI Extractor** | Peeks inside encrypted TLS handshakes to extract Server Name Indication — classifies domains without breaking encryption |
| 🚦 **Rule Engine** | Applies configurable filtering rules and generates detailed analytics reports |
| ⚡ **Concurrency Engine** | Multithreaded architecture with minimal lock contention for high-throughput packet processing |

<br/>

A **fully event-driven, cloud-native backend** on AWS with zero servers to manage. Every component is a managed service — wired together through events to form a complete data pipeline from ingestion to delivery.

| AWS Service | Role in the System |
|:------------|:------------------|
| ⚡ **Lambda** | Serverless compute — handles ingestion, processing, and API serving functions independently |
| 🔗 **API Gateway** | Exposes clean, versioned REST endpoints to clients with built-in throttling |
| 🗄️ **DynamoDB** | NoSQL storage engine for categorised and indexed news articles at any scale |
| ⏰ **EventBridge** | Cron-like scheduler that triggers the ingestion pipeline automatically |
| 📊 **CloudWatch** | Centralised logging, metrics, and alerting for every Lambda function |
| 🪣 **S3** | Raw data lake storing unprocessed news content before pipeline processing |

### 🛡️ 03 · AudioShield &nbsp;&nbsp; ![Status](https://img.shields.io/badge/In%20Development-7c6af7?style=flat-square)

</div>

<div align="center">

![React](https://img.shields.io/badge/React-0d0d14?style=for-the-badge&logo=react&logoColor=00d8ff)
![Flask](https://img.shields.io/badge/Flask-0d0d14?style=for-the-badge&logo=flask&logoColor=a89cf7)
![AES-GCM](https://img.shields.io/badge/AES--GCM-0d0d14?style=for-the-badge&logoColor=ff88ff)
![SHA-256](https://img.shields.io/badge/SHA--256-0d0d14?style=for-the-badge&logoColor=ff88ff)
![SQLite](https://img.shields.io/badge/SQLite-0d0d14?style=for-the-badge&logo=sqlite&logoColor=a89cf7)
![QR Code](https://img.shields.io/badge/QR%20Code-0d0d14?style=for-the-badge&logoColor=00ff9d)

</div>

<br/>

**Passwordless file sharing where your audio file is the cryptographic key.** No accounts. No passwords. No weak credentials. Just a file only you possess.

| Step | What Happens Under the Hood |
|:-----|:---------------------------|
| 🎵 **Upload** | User selects any file to protect + an audio file as the key |
| 🔑 **Key Derivation** | `SHA-256(audio_bytes)` derives a deterministic 256-bit encryption key from the raw audio |
| 🛡️ **Encryption** | AES-GCM encrypts the file using a freshly generated random 12-byte nonce |
| 📦 **Distribution** | Backend returns a unique Room Code and a scannable QR Code |
| 🔽 **Decryption** | Receiver uploads the same audio → same key derived → file unlocked |
| 🧹 **Auto Cleanup** | Server automatically purges all encrypted files after 24 hours |

> ❌ A **single bit change** in the audio file produces a completely different key — decryption fails immediately
