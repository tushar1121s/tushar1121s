### 🔬 Deep Packet Inspection Engine

</div>

![Java](https://img.shields.io/badge/Java-ff6b6b?style=flat-square&logo=openjdk&logoColor=white)
![Multithreading](https://img.shields.io/badge/Multithreading-4cffee?style=flat-square)
![Networking](https://img.shields.io/badge/Computer%20Networking-4cffee?style=flat-square)
![TLS](https://img.shields.io/badge/TLS%2FSNI-ff88ff?style=flat-square)
![PCAP](https://img.shields.io/badge/PCAP%20Parsing-ff88ff?style=flat-square)

A **multithreaded network analysis engine** built in Java that works on real network traffic files. Goes deep into packet-level data — not a toy project.

| Feature | Detail |
|:--------|:-------|
| 📦 **PCAP Parsing** | Reads and parses real network traffic capture files |
| 🔁 **Flow Tracking** | Stateful Five-Tuple hashing to track every connection |
| 🔐 **TLS SNI Extraction** | Classifies encrypted domains without breaking encryption |
| 🚦 **Rule Engine** | Applies configurable filters and generates analytics reports |
| ⚡ **Concurrency** | Multithreaded design with minimal lock contention |

### ☁️ Serverless News Aggregator

</div>

![AWS](https://img.shields.io/badge/AWS-ff9a20?style=flat-square&logo=amazonaws&logoColor=white)
![Lambda](https://img.shields.io/badge/Lambda-ff9a20?style=flat-square&logo=awslambda&logoColor=white)
![DynamoDB](https://img.shields.io/badge/DynamoDB-ff9a20?style=flat-square&logo=amazondynamodb&logoColor=white)
![S3](https://img.shields.io/badge/S3-ff9a20?style=flat-square&logo=amazons3&logoColor=white)
![EventBridge](https://img.shields.io/badge/EventBridge-ff9a20?style=flat-square&logo=amazonaws&logoColor=white)

A **fully event-driven, cloud-native backend** on AWS — zero servers to manage. Every piece of the pipeline is a managed service wired together with events.

| Service | Role |
|:--------|:-----|
| ⚡ **Lambda** | Serverless functions for ingestion, processing and serving |
| 🔗 **API Gateway** | Clean REST endpoints exposed to the outside world |
| 🗄️ **DynamoDB** | NoSQL storage for categorised articles at scale |
| ⏰ **EventBridge** | Scheduler that triggers the pipeline automatically |
| 📊 **CloudWatch** | Monitoring, logging and alerting |
| 🪣 **S3** | Raw data lake for unprocessed news content |

### 🛡️ AudioShield &nbsp; ![Status](https://img.shields.io/badge/In%20Development-7c6af7?style=flat-square)

</div>

![React](https://img.shields.io/badge/React-00d8ff?style=flat-square&logo=react&logoColor=white)
![Flask](https://img.shields.io/badge/Flask-a89cf7?style=flat-square&logo=flask&logoColor=white)
![AES-GCM](https://img.shields.io/badge/AES--GCM-ff88ff?style=flat-square)
![SHA256](https://img.shields.io/badge/SHA--256-ff88ff?style=flat-square)
![SQLite](https://img.shields.io/badge/SQLite-a89cf7?style=flat-square&logo=sqlite&logoColor=white)
![QR](https://img.shields.io/badge/QR%20Code-00ff9d?style=flat-square)

**Passwordless file sharing using audio files as cryptographic keys.** No accounts. No passwords. Just audio.

| Step | What Happens |
|:-----|:------------|
| 🎵 **Upload** | User uploads a file + an audio file |
| 🔑 **Key Gen** | `SHA-256(audio_bytes)` → 256-bit encryption key |
| 🛡️ **Encrypt** | AES-GCM with random 12-byte nonce encrypts the file |
| 📦 **Share** | Receiver gets a unique Room Code + QR Code |
| 🔽 **Decrypt** | Same audio → same key → file unlocked |
| 🧹 **Cleanup** | All files auto-delete after 24 hours |

> ❌ Even a **single bit change** in the audio file → decryption fails instantly
